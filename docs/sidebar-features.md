# 左侧目录区域功能规划

本文档整理 MD Watcher 左侧目录树区域计划增加的交互功能，用于指导后续开发。

## 设计目标

- 从"只读预览器"升级为"可用的本地 Markdown 管理工具"
- 保持界面简洁，优先实现高频操作
- 所有文件操作都通过后端 API 完成，前端仅负责交互

## 详细方案

### 1. 切换工作目录

**交互流程：**

1. 用户点击左侧顶部的"打开文件夹"按钮
2. 前端调用 `POST /api/select-folder`
3. 后端使用 `tkinter.filedialog.askdirectory()` 弹出系统文件夹选择对话框
4. 用户选择目录后，后端：
   - 停止旧的文件监控
   - 切换 `BASE_DIR`
   - 启动新的文件监控
   - 返回新的目录路径
5. 前端刷新目录树，状态栏更新当前路径

**异常处理：**

- 用户取消选择：返回 `{"canceled": true}`
- 选择的路径不存在或不可读：返回 400 错误

### 2. 新建文件 / 新建文件夹

**交互流程：**

1. 用户在目录或文件上右键，选择"新建文件"或"新建文件夹"
2. 前端调用 `POST /api/create`，参数：`type=file|dir`、`path=父目录路径`、`name=默认名称`
3. 后端创建文件或文件夹
4. 后端通过 WebSocket 广播 `created` 事件
5. 前端自动进入重命名状态

**默认名称规则：**

- 新建文件：`新建文档.md`，同名时自动加序号
- 新建文件夹：`新建文件夹`，同名时自动加序号

### 3. 重命名

**交互流程：**

1. 用户右键选择"重命名"或双击文件名（待定）
2. 文件名变为输入框，自动选中当前名称（不含扩展名）
3. 用户回车或失焦后，前端调用 `POST /api/rename`
4. 参数：`old_path`、`new_name`
5. 后端重命名文件，通过 WebSocket 广播 `moved` 事件
6. 前端刷新目录树，如果当前打开的文件被重命名，同步更新地址

### 4. 删除

**交互流程：**

1. 用户右键选择"删除"
2. 前端弹出确认框："确定要删除 `xxx.md` 吗？"
3. 用户确认后，调用 `DELETE /api/file?path=...`
4. 后端删除文件或文件夹
5. 通过 WebSocket 广播 `deleted` 事件
6. 如果删除的是当前打开的文件，右侧预览区显示占位提示

### 5. 刷新目录

**交互流程：**

1. 用户点击刷新按钮或右键选择"刷新"
2. 前端重新调用 `GET /api/tree`
3. 刷新目录树，保持当前展开状态和选中状态

---

## API 规划

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/select-folder` | 弹出系统对话框选择目录 |
| POST | `/api/create` | 新建文件或文件夹 |
| POST | `/api/rename` | 重命名文件或文件夹 |
| DELETE | `/api/file` | 删除文件或文件夹 |
| GET | `/api/tree` | 获取目录树（已存在） |

---

## 前端交互细节

- 右键菜单通过自定义 ContextMenu 组件实现，避免浏览器默认菜单
- 重命名输入框失焦或按回车提交，按 Esc 取消
- 删除操作必须二次确认
- 操作失败时通过右下角 Toast 提示错误信息
- 当前打开的文件在目录树中高亮显示
